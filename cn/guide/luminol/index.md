# 介绍
Luminol 是我们组织的第一个项目，最初创建的目的是寻求一个稳定且功能强大的 Folia 分支。

它包含以下功能：
 - 上游的 bug 修复
 - 有用的原版特性恢复
 - 来自其他分支的有用功能以及我们自己编写的功能

# 需求
根据 [Folia](https://docs.papermc.io/folia/faq/) 的原始文档，我们强烈建议您在拥有一个或多个具有 16 个以上物理核心的 CPU 时使用 folia 及其分支，否则，我们建议您使用非基于 Folia 的其他分支，如 Paper、Leaf 等。

此外，在 Luminol 中通常不推荐使用任何与原版特性相关的复杂红石机器或设施，因为它们可能无法正常工作。如果您正在寻找一个更适合原版体验的项目，请查看 [Lophine](https://github.com/LuminolMC/Lophine)，但由于 folia 重写了许多原版特性，因此原版特性的恢复仍然受到很大限制。

对于 Java 运行时，由于 Mojang 的变更，需要 Java 21 或更高版本。

# 提示
通常，Folia 及其分支的大多数用户都使用较大的堆内存，在这种情况下，我们建议您使用 ZGC 以减少游戏的暂停时间。此外，如果您使用 Linux，配置大页文件也是推荐的，这会对性能有明显的改善。

# 快速开始
要下载 Luminol，请查看 [发布](https://github.com/LuminolMC/Luminol/releases) 页面。

注意：我们不建议下载任何预发布版本，因为它们可能不稳定，并且可能包含会导致服务器崩溃或数据丢失的严重 bug。
